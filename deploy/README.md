## Quick Start (New Server)

### Prerequisites
- Ubuntu 22.04 or 24.04 LTS VM
- A non-root user (e.g. `leadflow`) with sudo access
- Git installed

### Step 1 — Clone & Run Setup
```bash
cd ~
git clone https://github.com/code4degree-oss/leadflow.git saas-project
cd saas-project/deploy
chmod +x setup.sh
sudo ./setup.sh
```

### Step 2 — Create Admin User
```bash
cd ~/saas-project/SAAS
source venv/bin/activate
DJANGO_SETTINGS_MODULE=config.settings.production python manage.py createsuperuser
```

### Step 3 — Verify
| Service   | Check Command                          |
|-----------|----------------------------------------|
| Gunicorn  | `sudo systemctl status gunicorn`       |
| PM2       | `pm2 status`                           |
| Nginx     | `sudo nginx -t`                        |
| Frontend  | Visit `http://YOUR_IP`                 |
| Admin     | Visit `http://YOUR_IP/leadflow-backend-admin/` |

## CI/CD Auto-Sync (GitHub Actions)

Once the server is set up, every `git push` to `main` automatically deploys changes.

### One-Time GitHub Setup
Go to **GitHub → Settings → Secrets and variables → Actions** and add:

| Secret Name       | Value                      |
|-------------------|----------------------------|
| `SERVER_HOST`     | Your VM's public IP        |
| `SERVER_USER`     | `leadflow`                 |
| `SERVER_PASSWORD` | Your VM SSH password       |

> If you use an SSH key (`.pem`) instead of a password, add `SERVER_SSH_KEY` with the key contents and leave `SERVER_PASSWORD` empty.

### How It Works
1. You push code to `main`
2. GitHub Actions SSHs into your VM
3. Runs `deploy/auto-sync.sh` which:
   - `git pull` latest code
   - Installs new Python/Node packages
   - Runs Django migrations
   - Rebuilds Next.js
   - Restarts Gunicorn & PM2

### Manual Trigger
You can also trigger a deploy manually from the GitHub **Actions** tab → **Deploy Auto-Sync Pipeline** → **Run workflow**.

## Configuration Reference

### `setup.sh` Variables (edit before first run)
| Variable           | Description                     | Default                  |
|--------------------|---------------------------------|--------------------------|
| `TARGET_USER`      | Linux user on the VM            | `leadflow`               |
| `PROJECT_DIR`      | Where the repo lives            | `/home/leadflow/saas-project` |
| `DOMAIN_NAME`      | Public IP or domain             | `20.18.160.17`           |
| `DB_NAME`          | PostgreSQL database name        | `leadflow_db`            |
| `DB_USER`          | PostgreSQL username             | `leadflow_user`          |
| `DB_PASSWORD`      | PostgreSQL password             | `leadflow123`            |
| `DJANGO_SECRET_KEY`| Django secret key               | *(change this!)*         |

### Environment Files (auto-generated by setup.sh)
| File                                              | Purpose                      |
|---------------------------------------------------|------------------------------|
| `SAAS/.env`                                       | Django configuration         |
| `leadflow-crm-frontend/leadflow/.env.local`       | Next.js API URL + Google Maps key |

## Troubleshooting

| Problem                    | Solution                                                |
|----------------------------|---------------------------------------------------------|
| **502 Bad Gateway**        | `sudo systemctl status gunicorn` — check if it crashed  |
| **Socket permission denied** | `sudo usermod -a -G leadflow www-data && sudo chmod 710 /home/leadflow && sudo systemctl restart nginx` |
| **Migrations fail**        | `sudo -u postgres psql -d leadflow_db -c "GRANT ALL ON SCHEMA public TO leadflow_user;"` |
| **Missing module**         | Activate venv and `pip install <module>`, then restart gunicorn |
| **Frontend not updating**  | `cd ~/saas-project/leadflow-crm-frontend/leadflow && npm run build && pm2 restart leadflow-frontend` |
